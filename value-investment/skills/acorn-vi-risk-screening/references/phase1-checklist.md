# 第一阶段：基础验证检查清单

## 数据来源

- [ ] 定量数据来自 `acorn vi query` API
- [ ] 定性数据来自 pdf2txt 转换的财报原文
- [ ] 未使用联网搜索、记忆数据、主观猜测

---

## 1. 言行一致验证

### 搜索命令

```bash
# 查找历年承诺
rg "將|会|計劃|预计|承诺" outputs/{公司}/*_an.txt --context 3

# 查找具体承诺类型
rg "研發 | 股东回报 | 資本開支 | 成本控制" outputs/{公司}/*_an.txt --context 5
```

### 检查清单

- [ ] 列出所有找到的承诺事项（不少于 3 项）
- [ ] 每项承诺都有原文引用（标注年份、页码）
- [ ] 获取对应年份的定量数据验证
- [ ] 计算达成率（实际/承诺）
- [ ] 标注验证等级（✅/🟡/⏳）

### 输出格式

```markdown
| 承诺事项 | 承诺年份 | 承诺原文 | 验证数据 | 实际执行 | 达成率 | 验证等级 |
|----------|----------|----------|----------|----------|--------|----------|
| 股东回报 | 2023 | "向股東提供可持續回報" | 股息 + 回购 | 1,440 亿 | 超额 | ✅ |
| 研发投入 | 2023 | "加大 AI 研發" | 一般行政开支 | +9% | 部分 | 🟡 |

言行一致评分：0.85/1.0
```

## 2. 风险提示验证

### 搜索命令

```bash
# 风险因素
rg "風險因素 | 風險" outputs/{公司}/*_an.txt --context 3

# 诉讼/担保
rg "訴訟 | 仲裁 | 擔保 | 抵押" outputs/{公司}/*_an.txt --context 3

# 关联交易
rg "關聯交易 | 關聯方" outputs/{公司}/*_an.txt --context 5
```

### 检查清单

- [ ] 搜索"风险因素"章节
- [ ] 统计风险章节页数
- [ ] 统计风险数量
- [ ] 统计量化风险数量
- [ ] 搜索诉讼/担保/关联交易
- [ ] 标注是否发现重大未披露事项

### 输出格式

```markdown
| 风险类型 | 是否披露 | 披露位置 | 是否量化 | 验证等级 |
|----------|----------|----------|----------|----------|
| VIE 架构 | ✅ | 第 X 页 | ❌ | 🟡 |
| 投资波动 | ✅ | 管理层讨论 | ✅ | ✅ |
| 重大诉讼 | ❌ 未搜索到 | - | - | ✅ (无) |

风险披露评分：0.8/1.0
```

## 4. 坦诚度验证

### 搜索命令

```bash
# 坏消息关键词
rg "下降 | 減少 | 虧損 | 下滑 | decline" outputs/{公司}/*_an.txt --context 3

# 对比不同年份篇幅
# 手动统计 2021（好年份）vs 2023（差年份）管理层讨论页数
```

### 检查清单

#### 4.1 坏消息披露
- [ ] 对比好坏年份管理层讨论篇幅
- [ ] 统计坏消息提及次数
- [ ] 检查是否使用模糊词汇

#### 4.2 归因清晰度
- [ ] 列出业绩下滑时管理层提到的原因
- [ ] 检查每个原因是否有数据支持
- [ ] 计算清晰归因比例

#### 4.3 数据透明度
- [ ] 使用数据透明验证结果

#### 4.4 风险充分性
- [ ] 使用风险提示验证结果

#### 4.5 对比公平性
- [ ] 检查是否同时披露 GAAP/非 GAAP
- [ ] 检查是否突出不利对比
- [ ] 检查是否使用一次性收益美化

### 输出格式

```markdown
| 维度 | 验证结果 | 评分 |
|------|----------|------|
| 坏消息披露 | 2023 年利润 -39%，讨论篇幅 +100% | 0.9 |
| 归因清晰度 | 投资收益有数据，游戏版号未量化 | 0.7 |
| 数据透明度 | 83% 关键指标单独披露 | 0.8 |
| 风险充分性 | 8 页 15 项风险，量化 33% | 0.7 |
| 对比公平性 | 同时披露 GAAP/非 GAAP | 0.8 |

坦诚度评分：0.785/1.0（加权平均）
```
