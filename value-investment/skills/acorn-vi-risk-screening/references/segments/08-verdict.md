## 定性部分：量化归因增强

### 归因对象

对第 2-7 段发现的**显著异常**进行量化归因：
- 净利润波动（变化额>20%）
- ROE 下降（变化>5pp）
- 现金流异常（现金流/利润<0.8 或>1.5）
- 其他显著异常指标

### 五步量化归因法

```
1. 量化异常 → 2. 识别原因 → 3. 原文摘录 → 4. 量化验证 → 5. 归因分析
```

#### 步骤 1：量化异常

**目标**：精确计算异常的**变化总额**

**输出格式**：
```markdown
| 年份 | 净利润 | 变化额 |
|------|--------|--------|
| 2024 | 1,941 亿 | +789 亿 |
| 2023 | 1,152 亿 | -1,096 亿 |
| 2021 | 2,248 亿 | - |

异常变化总额：2024 年较 2023 年增长 +789 亿
```

---

#### 步骤 2：识别原因

**目标**：列出管理层提到的**所有原因**（不遗漏）

**搜索命令**：
```bash
rg "利潤 | 盈利 | 投資收益" outputs/{公司}/*_an.txt --context 5
```

**输出格式**：
```markdown
管理层提到的原因：
1. 投资收益波动
2. 研发开支增加
3. 递延税项调整（2023 年一次性）
```

---

#### 步骤 3：原文摘录

**目标**：完整引用管理层对每个原因的说明

**输出格式**：
```markdown
**原因 1：投资收益波动**
> "投資收益╱（虧損）淨額及其他 4,187 (-6,090)"
> 来源：综合收益表，第 123 页

**原因 2：研发开支增加**
> "一般及行政開支同比增長 9% 至人民幣 1,127 億元"
> 来源：管理层讨论及分析，第 7 页
```

---

#### 步骤 4：量化验证

**目标**：用定量数据验证每个原因，计算**贡献比例**

**核心公式**：
```
贡献额 = 原因对应的财务数据变化
贡献比例 = 贡献额 / 异常变化总额 × 100%
```

**输出格式**：
```markdown
| 原因 | 定量数据 | 贡献额 | 贡献比例 | 验证等级 |
|------|----------|--------|----------|----------|
| 1. 投资收益波动 | +419 亿 vs -609 亿 | +1,028 亿 | 130.3% | ✅ 完全验证 |
| 2. 研发开支增加 | 1,127 亿 vs 1,035 亿 | -92 亿 | -11.7% | 🟡 部分验证 |
| 3. 递延税项调整 | 未单独披露 | 无法计算 | ? | ⏳ 待验证 |

计算过程：
- 投资收益：419 - (-609) = +1,028 亿
- 贡献比例：1,028 / 789 = 130.3%
```

**验证等级标准**：
| 等级 | 标记 | 标准 |
|------|------|------|
| 完全验证 | ✅ | 数据完全一致，可精确计算贡献额 |
| 部分验证 | 🟡 | 方向一致但细节缺失（如未单独披露） |
| 无法验证 | ❌ | 数据矛盾或未披露 |
| 待验证 | ⏳ | 需查阅其他章节/附注 |

---

#### 步骤 5：归因分析

**目标**：综合评估管理层解释的可信度

**核心公式**：
```
已解释比例 = 所有已验证原因的贡献比例之和
无法验证比例 = 100% - 已解释比例
```

**输出格式**：
```markdown
已解释部分：
- 投资收益波动：+1,028 亿（130.3%）
- 研发开支增加：-92 亿（-11.7%）
- 小计：+936 亿（118.6%）

无法解释部分：
- 递延税项调整：数据未披露
- 小计：无法计算

归因完整性：
- 已验证原因贡献：118.6%（>100%，已完全解释）
- 剩余未解释：0%

管理层可信度：🟢 高
```

**归因完整性判定标准**：
| 已解释比例 | 无法验证部分 | 管理层可信度 | 风险判定 |
|------------|--------------|--------------|----------|
| ≥100% | 无 | 🟢 高 | 🟢 通过 |
| 80-99% | 小（<20%） | 🟡 中 | 🟡 需关注 |
| 50-79% | 中（20-50%） | 🟠 低 | 🟠 高风险 |
| <50% | 大（>50%） | 🔴 极低 | 🔴 排除 |

详见 [references/phase2-attribution.md](references/phase2-attribution.md)
