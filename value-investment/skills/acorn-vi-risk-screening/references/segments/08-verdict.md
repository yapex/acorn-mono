# 第八段：综合判定

基于第二至七段的分析结果，得出六维风险判定和投资分层。

---

## 1. 需要的数据

### 数据来源

汇总以下各段的判定结果：

| 段 | 内容 | 关键输出 |
|----|------|----------|
| 第二段 | 偿债能力 | 综合判定（✅/⚠️/🔴） |
| 第三段 | 组合信号 | 造假风险（✅/⚠️/🔴）+ 触发数量 |
| 第四段 | 极值排查 | P0/P1/P2 风险（✅/⚠️/🔴） |
| 第五段 | 长期价值 | 评分（0-100） |
| 第六段 | 时间序列 | 红灯数量 |
| 第七段 | 特殊项目 | 综合判定（✅/⚠️/🔴） |

---

## 2. 展示样例

### 六维风险判定

| 维度 | 段 | 结果 | 否决 |
|------|-------|:----:|:----:|
| 造假风险 | 第三段 | 🔴 高风险 | ✅ 否决 |
| 生存风险 | 第二段 | ✅ 安全 | — |
| 经营恶化风险 | 第四段 | ⚠️ 关注 | — |
| 资金链风险 | 第二段 | ✅ 安全 | — |
| 投资风险 | 第六段 | ⚠️ 红灯3项 | — |
| 长期价值 | 第五段 | 🟢 85分 | — |

### 投资分层

```
┌─────────────────────────────────────────┐
│           造假风险：🔴 高风险            │
│              ↓ 直接排除                  │
│         ❌ 不适合投资                    │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│           生存风险：🔴 危险              │
│              ↓ 直接排除                  │
│         ❌ 不适合投资                    │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│         长期价值：🔴 <60分              │
│              ↓ 不适合长期持有            │
│         ⚠️ 短期投机需谨慎               │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│     造假✅ + 生存✅ + 价值≥80           │
│              ↓ 全部通过                  │
│         🟢 适合长期持有                 │
└─────────────────────────────────────────┘
```

### 综合判定矩阵

| 造假 | 生存 | 价值 | 投资建议 |
|:----:|:----:|:----:|----------|
| 🔴 | * | * | ❌ 直接排除 |
| * | 🔴 | * | ❌ 直接排除 |
| ✅ | ✅ | <60 | ⚠️ 不适合长期持有 |
| ✅ | ✅ | 60-79 | 🟡 可关注，有瑕疵 |
| ✅ | ✅ | ≥80 | 🟢 适合长期持有 |

---

## 3. 核查结果

### 否决规则

| 规则 | 触发条件 | 结果 |
|------|----------|------|
| 造假否决 | 第三段 🔴 | 直接排除 |
| 生存否决 | 第二段 🔴 | 直接排除 |
| 价值否决 | 第五段 <60 | 不适合长期持有 |

### 判定输出

```
综合判定：❌ 不适合投资

否决原因：
- 🔴 造假风险：第三段触发C01+C03+C04

通过项：
- ✅ 生存风险：通过
- ✅ 资金链：通过

否决项：
- 🔴 造假风险：高风险

结论：财务造假嫌疑，不适合投资
```

## 5. 快速判定

如果时间有限，可用简化流程：

```
1. 造假风险 🔴？ → 是 → ❌ 直接排除
2. 生存风险 🔴？ → 是 → ❌ 直接排除
3. 造假✅ 且 生存✅ 且 价值≥80？ → 是 → 🟢 适合长期持有
4. 造假✅ 且 生存✅ 且 价值60-79？ → 是 → 🟡 可关注
5. 其他情况 → ⚠️ 不适合长期持有
```
