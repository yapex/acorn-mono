# 第七段：特殊项目分析 + 定性验证

## 定量部分：特殊项目检查

### 7.1 审计意见

**数据来源**：pdf2txt 财报原文

**搜索命令**：
```bash
rg "審計意見 | 审计师 | 核數師|audit" outputs/{公司}/*_an.txt --context 5
```

**检查项**：
- [ ] 审计师事务所名称
- [ ] 审计意见类型（标准无保留/保留意见/无法表示意见）
- [ ] 关键审计事项

**判定标准**：
| 意见类型 | 风险等级 | 行动 |
|----------|----------|------|
| 标准无保留 | 🟢 低 | 通过 |
| 带强调事项段 | 🟡 中 | 需关注 |
| 保留意见 | 🟠 高 | 谨慎 |
| 无法表示意见 | 🔴 极高 | 排除 |

---

### 7.2 股东回报

**数据来源**：`acorn vi query` + pdf2txt 财报原文

**搜索命令**：
```bash
rg "股息 | 分紅 | 回購|dividend|buyback" outputs/{公司}/*_an.txt --context 5
```

**检查项**：
- [ ] 现金股息金额
- [ ] 股份回购金额
- [ ] 分红率（股息/净利润）
- [ ] 分红连续性（是否连续 5 年分红）

**判定标准**：
| 分红率 | 连续性 | 风险等级 |
|--------|--------|----------|
| ≥30% | ≥5 年 | 🟢 优秀 |
| 10-29% | 3-4 年 | 🟡 良好 |
| <10% | 1-2 年 | 🟠 一般 |
| 0% | 0 年 | 🔴 无回报 |

---

## 定性部分：基础验证增强

### 7.5 言行一致验证

**目标**：验证管理层承诺是否兑现

**方法**：对比历年承诺原文 vs 后续年份执行数据

**输出格式**：
```markdown
| 承诺事项 | 承诺年份 | 承诺原文 | 验证数据 | 实际执行 | 达成率 | 验证等级 |
|----------|----------|----------|----------|----------|--------|----------|
| 股东回报 | 2023 | "向股東提供可持續回報" | 股息 + 回购 | 1,440 亿 | 超额 | ✅ |
| 研发投入 | 2023 | "加大 AI 研發" | 一般行政开支 | +9% | 部分 | 🟡 |

言行一致评分：0.85/1.0
```

**评分标准**：
| 达成率 | 评分 |
|--------|------|
| ≥90% | 0.9-1.0 |
| 70-89% | 0.7-0.89 |
| <70% | <0.7 |

详见 [references/phase1-checklist.md](references/phase1-checklist.md)

### 7.6 风险提示验证

**目标**：检查重大风险是否充分披露

**搜索命令**：
```bash
rg "風險 | 訴訟 | 擔保 | 關聯交易" outputs/{公司}/*_an.txt --context 3
```

**输出格式**：
```markdown
| 风险类型 | 是否披露 | 披露位置 | 是否量化 | 验证等级 |
|----------|----------|----------|----------|----------|
| VIE 架构 | ✅ | 第 X 页 | ❌ | 🟡 |
| 投资波动 | ✅ | 管理层讨论 | ✅ | ✅ |
| 重大诉讼 | ❌ 未搜索到 | - | - | ✅ (无) |

风险披露评分：0.8/1.0
```

### 7.7 数据透明验证

**目标**：检查关键指标是否单独披露

**检查项**：
- [ ] 研发开支是否单独披露
- [ ] 投资收益是否单独披露
- [ ] 员工成本是否单独披露
- [ ] 商誉减值是否单独披露
- [ ] 分业务收入是否单独披露

**输出格式**：
```markdown
| 指标 | 是否单独披露 | 披露位置 | 透明度 |
|------|--------------|----------|--------|
| 研发开支 | ❌ | 含在一般行政开支中 | 🔴 |
| 投资收益 | ✅ | 综合收益表 | 🟢 |

透明度评分：0.8/1.0（80% 单独披露）
```

---

### 7.8 坦诚度验证

**目标**：评估管理层披露的坦诚程度

**五个维度**：

| 维度 | 验证方法 | 权重 |
|------|----------|------|
| 坏消息披露 | 对比好坏年份讨论篇幅 | 25% |
| 归因清晰度 | 原因是否有数据支持 | 25% |
| 数据透明度 | 关键指标单独披露比例 | 25% |
| 风险充分性 | 风险章节长度 + 量化比例 | 15% |
| 对比公平性 | GAAP/非 GAAP 披露 | 10% |

**输出格式**：
```markdown
| 维度 | 验证结果 | 评分 |
|------|----------|------|
| 坏消息披露 | 2023 年利润 -39%，讨论篇幅 +100% | 0.9 |
| 归因清晰度 | 投资收益有数据，游戏版号未量化 | 0.7 |
| 数据透明度 | 83% 关键指标单独披露 | 0.8 |
| 风险充分性 | 8 页 15 项风险，量化 33% | 0.7 |
| 对比公平性 | 同时披露 GAAP/非 GAAP | 0.8 |

坦诚度评分：0.785/1.0（加权平均）
```

详见 [references/phase1-checklist.md](references/phase1-checklist.md)
